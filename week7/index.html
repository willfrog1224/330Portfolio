~11~

functions can have properties and methods

Call() - set the value of this

description property - descibes what it does

memoization - result caching

Immediately Invoked Function Expression or IIFE - function that is invoked as soon as it is defined, a useful way to perform tasks but keep the vaiables in the scope fo the function

No way to remove a variable from a scope once it is declared

IIFE can be used to set up any initialization code that there will be no need for again

IIFE can be used to enclose a block of code inside its own private scope so it does not affect the other parts of the program

recursive function - invokes itself until a certain condition is met

Callbacks can be used to facilitate event-driven asynchronous programming

Callback hell is the term used to refer to this tangled mess of code

A promise represents the future result of an asynchronous operation

Resolved ― the asynchronous operation was completed successfully.

Rejected ― the asynchronous operation did not work as expected, wasn't successfully completed or resulted in an error.

Once a promise has been settled, the then() method can be used to deal with the outcome

Promises come into their own when multiple asynchronous tasks are required to be carried out one after the other.

Async functions were added to the ES2017 specification.

Callbacks can be used to build more generalized functions

functions can accept another function as an argument (a callback), but they can also return a function

closure is formed when the inner function is returned by the outer function, maintaining access to any variables declared inside the enclosing function

closure is formed when a function returns another function that then maintains access to any variables created in the original function scope

A pure function is a function that adheres to the following rules:

1) return value only depends on provided arguments

2) no side-effects

3) referential transparency

pure function must have:

At least one argument

A return value

Higher-order functions are functions that accept another function as an argument, or return another function as a result, or both

Currying - involves the partial application of functions, when not all arguments have been supplied to the function, so it returns another function that retains the arguments already provided, and expects the remaining arguments that were omitted when the original function was called

~13~

Ajax - a technique that allows web pages to communicate asynchronously with a server, and it dynamically updates web pages without reloading

Web pages were no longer static, but dynamic applications

web of computers known as the internet can be separated into two parts: clients and servers

Asynchronous - can run while data is sent back and forth

Ajax enabled JavaScript to send requests and receive responses from a server, allowing content to be updated in real time.

Fetch API introduced the Response interface that deals with the object that is returned when the promise is fulfilled

redirect() method can be used to redirect to another URL

text() method takes a stream of text from the response, reads it to completion and then returns a promise that resolves to a USVSting object that can be treated as a string in JavaScript

blob() method - used to read a file of raw data, such as an image or a spreadsheet

JSON is probably the most common format for AJAX responses

json() method is used to deal with these by transforming a stream of JSON data into a promise that resolves to a JavaScript object.

five most commonly used verbs when dealing with the web are:

GET requests to retrieve resources

POST requests, usually used to create a resource but can actually perform any task

PUT requests to upsert, which means insert a resource or update it entirely

PATCH requests to make partial updates to a resource

DELETE requests to delete a resources.

HTTP headers are used to pass on any additional information about a request or response

We can also use Ajax to send information

Fetch API includes the FormData interface, which makes it much easier to submit information in forms using Ajax
