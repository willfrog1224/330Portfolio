Testing and Debugging

System error ― a problem with the system or external devices with which the program is interacting

Programmer error ― the program contains incorrect syntax or faulty logic; it could even be as simple as a typo

User error ― the user has entered data incorrectly, which the program is unable to handle

exception is an error that produces a return value that can then be used by the program to deal with the error.

stack trace - a sequence of functions or method calls that lead to the point where the error occurred

warning can occur if theres an error in the code that isn't enough to cause the program to crash

Linting tools such as JS Lint,JS Hint, and ES Lint can be used to test the quality of JavaScript code, beyond simply using strict mode

most basic form of debugging is to use the alert() method to show a dialog at certain points in the code

Most modern JavaScript environments have a console object that provides a number of methods for logging information and debugging

modern browsers also have a debugging tool that allows you to set breakpoints in your code that will pause it at certain points.

Remember to remove any references to the debugger command before shipping any code, otherwise the program will appear to freeze when people try to use it!
